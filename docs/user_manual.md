# PtLPOJ User Manual

欢迎使用 **PtLPOJ (Personal Local Python Online Judge)**，这是一款专为内部团队和教学场景打造的极轻量、高安全的代码评测平台。本手册将指引您如何在 VS Code 中沉浸式地刷题与发布测试。

---

## 👨‍🎓 学生/选手指南 (Student Guide)

作为刷题者，您的全部操作都将在您最熟悉的 IDE —— **Visual Studio Code** 中完成，无需打开任何网页。

### 1. 登录系统 (Authentication)
本系统采用无密码的极客登录方式：
1. 在 VS Code 中安装 PtLPOJ 插件。
2. 按 `Ctrl+Shift+P` (或 macOS 下的 `Cmd+Shift+P`) 呼出命令面板，输入 `PtLPOJ: Login`。
3. 输入您被系统管理员加白的邮箱地址。
4. 去邮箱 (如果在内测阶段，可能直接由管理员提供) 查收 6 位数一次性动态密码 (OTP，5 分钟内有效)。
5. 将 OTP 填入弹出的输入框即可。成功后，底部状态栏会显示 `PtLPOJ: Logged In`。

### 2. 浏览与拉取题目 (Browse & Fetch)
登录后，您会在 VS Code 左侧边栏看到一个名为 `PtLPOJ` 的图标（或是展开的 `Problem Sets` 视图面板）。
* 这里会列出所有可用的题目以及您当前的进度情况：已通过(✅)、未通过(❌) 或未尝试(⚪)。
* **双击某个题目**，系统会自动拦截将其元数据拉取到您的本地配置目录下，并为您**自动打开两个分页**：
    * 右侧：题目的 Markdown 题面描述。
    * 左侧：一段已经写好函数签名（`Scaffold`）的 Python 代码文件。

### 3. 本地开发与无缝提交 (Develop & Submit)
PtLPOJ 最大的优势是你处于真正的本地环境中！
* 你可以直接利用电脑上安装的 Python 解释器运行该文件进行本地 Debug。
* 当你认为逻辑实现无误时，确保你的光标焦点在该 Python 代码文件中，直接按下快捷键 **`Ctrl+Alt+S`** (Mac 为 `Cmd+Alt+S`)。
* 代码会被一键自动上传至内网的安全沙盒中。

### 4. 查看判题结果 (Results)
提交后，系统会进行异步判定。您可以继续写其他代码（不会阻塞编辑器）。
当服务端跑完所有的隐藏测试用例后，你的 VS Code 侧边框会自动刷新，同时右下角会弹出详细结果：
* **AC (Accepted)**：代码逻辑完美通过，执行时间和内存也均被记录！
* **WA (Wrong Answer)**：在某一个隐藏测试点输出错误。为了防止“暴力枚举打表”，系统只会提示您失败于第几个用例，**不会下发具体的测试数据**。
* **TLE (Time Limit Exceeded)**：代码超过了预设的时间。
* **RE / OLE**：运行时异常 / 输出超限（如进入了打印死循环）。

## 👩‍🏫 教师/管理员指南 (Admin Guide)

管理员不仅享有普通用户的刷题权限，还肩负着为系统补充高质量题库的任务。

### 1. 关于“隐藏测试点”与防作弊机制
*学生端点击运行题目时，插件从服务端拉取的 Python 模板（Scaffold）中，仅仅包含供理解的 Demo 用例。真正用于判题的数百条、覆盖边界条件的数据 (`tests.txt`) 是被绝对隔离保存在服务端宿主机上的。*

即使学生在提交的文件中恶意植入如：
```python
import os
os.system("cat tests.txt > my_answer.txt")
```
的系统调用企图偷取答案，由于服务端在底层拉起判定容器时，对学生代码的上下文启用了完全的读写隔离与降权，这种越权行为将被 Docker Native 拦截而触发异常。

### 2. (功能预览) 题库的动态维护
*目前在项目早期阶段，增加题目需要依赖系统维护者直接放置文件到服务器特定的目录中。*

但在接下来的 **Phase 8** 更新完毕后，系统将正式下发具有图形化和更高执行抽象的管理脚本：
1. 您将会被系统注册为 `Teacher` Role。
2. 您可以通过 API 提交一个新的 JSON 或 Markdown。系统底层会自动同步进主 SQLite 数据库并绑定多维 `Tag` 标签，学生端的 TreeView 即时可见。
3. 如果您发现某个测试用例存在漏洞，您也可以随时上传修正后的用例文件热覆写，所有的再次评判将采用新数据基准。

*(注：系统具体的批量操作脚本或面板，请参阅随后上线的 `Admin Tools` 文档章节)。*
