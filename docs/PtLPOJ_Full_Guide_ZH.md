# PtLPOJ 全量部署与使用指南 (v0.2.x+)

欢迎使用 **PtLPOJ (Ptlantern's Learning Platform Online Judge)**！
本文档旨在帮助团队管理员、教师及普通用户快速搭建 PtLPOJ 环境，并利用 VS Code 插件进行沉浸式编程练习。

---

## 🛠️ 第一部分：后端服务器部署 (Server Deployment)

服务端承担着 API 网关、用户鉴权与真正的 Docker 安全沙盒评测任务。

### 1.1 环境要求
- **系统**: 推荐 Ubuntu 20.04/22.04 LTS (必须是 Linux，以支持 Docker cgroups)。
- **Docker**: 必须安装 Docker Engine。
- **Go**: 如需自行编译，需 Go 1.21+。

### 1.2 快速部署步骤
1. **拉取代码**:
   ```bash
   git clone https://github.com/LynPtl/PtLPOJ.git
   cd PtLPOJ/server
   ```
2. **初始化沙盒镜像**:
   ```bash
   docker pull python:3.11-alpine
   ```
3. **编译并启动**:
   ```bash
   go build -o ptlpoj-server main.go
   
   # 启动服务（默认监听 8080 端口）
   ./ptlpoj-server
   ```
   *注意：如果要在非本机环境下使用，请确保服务器防火墙已放行 8080 端口。*

## 🔌 第二部分：VS Code 扩展插件配置 (Extension Setup)

现在插件已经支持动态配置服务器地址，不再锁定于 `localhost`。

### 2.1 获取插件
- **方式 A (推荐)**: 访问项目的 GitHub 仓库，在 **Actions** 页面找到最新的 `Package Extension` 任务，下载 `ptlpoj-vsix` 产出物。
- **方式 B (手动编译)**:
  ```bash
  cd client
  npm install
  npm run compile
  npx @vscode/vsce package --out ptlpoj-client.vsix
  ```

### 2.2 安装与配置
1. 在 VS Code 中，通过 `Extensions -> ... -> Install from VSIX...` 安装生成的插件包。
2. **连接自有后端**:
   - 打开 VS Code 设置 (`Ctrl+,`)。
   - 搜索 `ptlpoj.serverUrl`。
   - 将其修改为您部署的后端地址。例如：`http://192.168.1.100:8080/api`。
   > [!IMPORTANT]
   > 必须以 `/api` 结尾，且确保协议头 (http/https) 正确。

## 📖 第三部分：用户使用手册 (User Manual)

### 3.1 沉浸式登录
1. 点击左侧活动栏的 **PtLPOJ** 图标。
2. 在未登录状态下，点击题库上方的 **“$(sign-in) 点击登录 PtLPOJ”**。
3. 在弹出的 Webview 界面中输入邮箱并验证（验证码目前在服务端控制台打印）。
4. 登录成功后，系统将自动跳转至 **Dashboard**。

### 3.2 练习与提交
1. 从左侧侧边栏选择题目，双击打开。
2. 在生成的 `.py` 文件中编写逻辑。
3. 点击编辑器顶部的 **CodeLens** 按钮：
   - **Run Local Test**: 在本地运行基础测试。
   - **Submit to Sandbox**: 提交至云端进行全量隐藏测试验证。
4. 结果将通过右下角通知并在侧边栏动态更新状态（✅ AC / ❌ WA 等）。

## ❓ 常见问题 (FAQ)

- **Q: 为什么侧边栏显示获取题目失败？**
  - A: 请检查 `ptlpoj.serverUrl` 设置是否正确，以及后端服务是否已启动并可跨网段访问。
- **Q: 注册不了账号怎么办？**
  - A: 目前版本需要管理员在服务端的 SQLite 数据库 `users` 表中手动插入邮箱地址作为白名单。
- **Q: 打包失败提示 Node 版本问题？**
  - A: 请升级本地 Node.js 至 v20 或更高版本，这是 `vsce` 的强制要求。
