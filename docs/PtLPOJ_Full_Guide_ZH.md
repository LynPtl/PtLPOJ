# PtLPOJ 全量部署与使用指南 (v0.2.x+)

欢迎使用 **PtLPOJ (Ptlantern's Learning Platform Online Judge)**！
本文档旨在帮助团队管理员、教师及普通用户快速搭建 PtLPOJ 环境，并利用 VS Code 插件进行沉浸式编程练习。

## 🛠️ 第一部分：后端服务器部署 (Server Deployment)

服务端承担着 API 网关、用户鉴权与真正的 Docker 安全沙盒评测任务。

### 1.1 环境要求
- **操作系统**: 推荐 Ubuntu 20.04/22.04 LTS (必须是 Linux，系统依赖 Docker cgroups 进行资源硬隔离，不推荐在 Windows Server / macOS 直接部署跑评测)。
- **Docker**: 必须安装 Docker Engine。请确保运行后端程序的用户已加入 `docker` 组 (即可以无 `sudo` 执行 `docker ps`)。
  ```bash
  curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh get-docker.sh
  sudo usermod -aG docker $USER # 注销并重新登录生效
  ```
- **Go 环境**: 如需自行编译，需 Go 1.21+。
- **C 编译器**: 编译需要 CGO 支持 (底层依赖 SQLite3)，请预先安装 `gcc` 核心库包 (如 `sudo apt-get install -y gcc build-essential`)。

### 1.2 快速部署步骤
1. **拉取代码**:
   ```bash
   git clone https://github.com/LynPtl/PtLPOJ.git
   cd PtLPOJ/server
   ```
2. **初始化沙盒镜像**:
   ```bash
   docker pull python:3.11-alpine
   ```
3. **编译并启动**:
   ```bash
   # 若未安装 gcc，请执行 sudo apt install gcc
   go mod tidy
   CGO_ENABLED=1 go build -o ptlpoj-server main.go
   
   # 可选：配置环境变量 (端口、JWT 秘钥等) 并启动服务
   export PTLPOJ_PORT=8080
   export JWT_SECRET="your-super-strong-jwt-secret"
   export PTLPOJ_INIT_ADMIN_EMAIL="admin@yourcollege.edu" # 启动时将该邮箱注入白名单
   ./ptlpoj-server
   ```
   *注意：如果要在非本机环境下使用，请确保服务器防火墙已放行对应端口（默认 8080）。启动成功后会输出 SQLite 连接及 Sandbox 初始化完成的标识。*

## 🔌 第二部分：VS Code 扩展插件配置 (Extension Setup)

现在插件已经支持动态配置服务器地址，不再锁定于 `localhost`。

### 2.1 获取插件
- **方式 A (推荐)**: 访问项目的 GitHub 仓库，在 **Actions** 页面找到最新的 `Package Extension` 任务，下载 `ptlpoj-vsix` 产出物。
- **方式 B (手动编译)**:
  ```bash
  cd client
  npm install
  # 利用自动化流以规避交互提示打包插件
  echo "y" | npx @vscode/vsce package --out ptlpoj-client.vsix
  ```

### 2.2 安装与配置
1. **在 VS Code 中安装**: 此时 `client` 目录下会生成 `ptlpoj-client.vsix` 文件。在您的 VS Code 左侧扩展栏点击右上方的 `...` -> `Install from VSIX...`，选择该文件进行安装。
2. **连接自有后端**:
   - 打开 VS Code 设置 (`Ctrl+,`)。
   - 搜索 `ptlpoj.serverUrl`。
   - 将其修改为您部署的后端地址。例如：`http://192.168.1.100:8080/api`。
   > [!IMPORTANT]
   > 必须以 `/api` 结尾，且确保协议头 (http/https) 正确。

## 📖 第三部分：用户使用手册 (User Manual)

### 3.1 沉浸式登录
1. 点击左侧活动栏的 **PtLPOJ** 图标。
2. 在未登录状态下，点击题库上方的 **“$(sign-in) 点击登录 PtLPOJ”**。
3. 在弹出的 Webview 界面中输入邮箱并验证（验证码目前在服务端控制台打印）。
4. 登录成功后，系统将自动跳转至 **Dashboard**。

### 3.2 练习与提交
1. 从左侧侧边栏选择题目，双击打开。
2. 在生成的 `.py` 文件中编写逻辑。
3. 点击编辑器顶部的 **CodeLens** 按钮：
   - **Run Local Test**: 在本地运行基础测试。
   - **Submit to Sandbox**: 提交至云端进行全量隐藏测试验证。
4. 结果将通过右下角通知并在侧边栏动态更新状态（✅ AC / ❌ WA 等）。

## 👑 第四部分：管理员控制台 (Admin Panel)

如果您是教师或团队管理员，PtLPOJ 提供了直接嵌在 VS Code 中的可视化操作面板，供您极速管理题库与白名单。

### 4.1 管理员口令配置
后端启动时，系统依赖环境变量中的 `PTLPOJ_ADMIN_TOKEN` 鉴权：
- **默认体验**：如果您什么都不配置直接启动 `go run main.go`，它将自动使用默认口令 `ptlpoj_default_admin`。
- **生产环境**：强烈建议在服务器启动时挂载口令，例如：`PTLPOJ_ADMIN_TOKEN=MySuperSecret ./ptlpoj-server`。

### 4.2 进入后台面板
1. **前置条件**：由于系统启用了高级的**“双盲验证”** (User JWT + Admin Token)，您必须首先在插件侧边栏 **登录您的普通用户账号**，然后再打开管理控制台（未登录会被防火墙直接拦截）。
2. 在 VS Code 按下 `Ctrl+Shift+P`，搜索并执行命令 `PtLPOJ: Open Admin Panel`。
3. 首次进入时，顶部会提示您输入上文配置的 **Admin Token**（储存于本地底层加密链，后续无需重复输入）。如需更换口令，可点击控制台右上角的 `🔄 Reset Admin Token`。
4. 验证成功后即可看到管理大屏。

### 4.3 核心特权功能
- **👥 用户白名单批量管理**：
  - **批量导入**：在输入框中键入用户邮箱。支持从外部试算表（如 Excel）批量复制（允许逗号、空格、换行符分隔），系统将自动执行并发分批导入。
  - **批量移除**：点击输入框右侧的 `Bulk Delete Users`，即可一键剔除输入的全部账户。
- **📚 题库自动化解析上传**：无需手动维护单独的 Markdown 描述和独立测试用例。点击控制台中的 `Select .py Files & Upload Bulk` 按钮。
  - 支持通过操作系统的原生文件选择器进行**多选（如按住 Ctrl 批量框选）上传多份 Python 源文件**。
  - 文件中需包含标准化的 `docstring` 与 `doctest` 断言，后端的 AST 解析器将自动分解代码脚手架、生成题目描述并入库。

## ❓ 常见问题与高级运维 (FAQ & Basic Ops)

### 运维与数据备份 (Operations)
- **SQLite WAL 机制与备份**：PtLPOJ 为了支持高并发回写判定状态已强制开启 WAL 模式。您的数据目录下会存在三个核心文件 (`ptlpoj_dev.db`, `*.db-wal`, `*.db-shm`)。**备份时请务必同时拷贝这三个文件或在服务停止时拷贝**以防数据损坏。
- **应用更新与数据迁移**：在从 `v0.2.x` 升级到 `v0.3.x` 以后，从 Markdown 同步题目的逻辑将被移除。管理员请使用 `PtLPOJ: Open Admin Panel` 中的可视化页面自动解析、同步 `.py` 新题目档。

### 排错 (Troubleshooting)

| 症状 / 报错信息 | 常见原因 | 解决办法 |
| :--- | :--- | :--- |
| **"Cannot connect to the Docker daemon"** | 运行 Server 的用户没有 Docker 操作权限，或 Docker Engine 尚未启动。 | 检查 `systemctl status docker`，并使用 `sudo usermod -aG docker $USER` 提权。 |
| **"bind: address already in use"** | 设定的端口 (默认 8080) 已被其他进程占用。 | 使用 `lsof -i:8080` 杀死占用进程，或使用 `export PTLPOJ_PORT=新端口` 启动。 |
| **客户端评测一直处于 "PENDING" 卡死** | Server 崩溃，或 Docker 守护进程宕机导致并发协程信道阻塞。 | 重启 `./ptlpoj-server`。系统带有奔溃恢复机制，会自动从数据库中拉起遗留为 PENDING 或 RUNNING 的孤儿判题。 |
| **侧边栏显示获取题目失败** | 后端服务未响应。 | 请检查 `ptlpoj.serverUrl` 设置是否正确，以及后端服务是否已启动并处于同一网络下。|
| **学生注册不了账号，提示不在白名单怎么办？** | 系统为邀请制。 | 请拥有权限的教师在 `PtLPOJ: Open Admin Panel` 中，将其邮箱加入白名单即可。|
| **打开管理面板点上传没反应？** | 客户端扩展版本过低。 | 请确保插件版本升级至 `v0.3.1+`，该版本支持底层的原生批量系统对话框。|
| **打包失败提示 Node 版本问题？** | Node 运行时落后。 | 请升级本地 Node.js 至 v20 或更高版本，这是 `vsce` 的强制机制。|
