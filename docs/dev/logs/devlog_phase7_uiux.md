# Phase 7 开发日志：UI/UX 深度优化与功能重构

## 1. 阶段概述
Phase 7 的核心目标是将 PtLPOJ 从“功能原型”转化为“生产级”集成开发环境扩展。本阶段摒弃了原生 Markdown 预览的局限性，通过 Webview 技术实现了富文本渲染、实时反馈迭代及个性化数据面板。

## 2. 核心里程碑

### 2.1 快速交互入口 (CodeLens) (v0.1.1)
- **挑战**：用户频繁在代码与侧边栏之间切换点击，操作链路过长。
- **方案**：在 Python 解题代码顶部动态注入虚拟链接。
- **成果**：实现“一键本地测试”与“一键提交沙盒”，显著降低用户操作心智负担。

### 2.2 重构题目视图 (Enriched ProblemView) (v0.1.2 - v0.1.4)
- **改进**：引入 `markdown-it` + `KaTeX` 渲染引擎。
- **特性**：全面支持 LaTeX 公式、GFM 高级表格与代码高亮。
- **反馈闭环**：将评测反馈从琐碎的任务栏弹窗整合为题目顶部的 **Sticky 进度条**，确保用户在滚动查看题面时仍能感知后台评测进度。

### 2.3 个人中心与统计 (Dashboard) (v0.1.5 - v0.1.9)
- **目的**：提供刷题进度的全局视角。
- **实现**：新增 `/api/user/stats` 接口，聚合存储于 SQLite 的提交历史。
- **视觉**：采用卡片式布局。为了优化用户反馈，将“新人指导”置顶并移除无关的每日任务横幅，使界面更聚焦于个人产出。

### 2.4 题库检索增强 (Filtering & Search) (v0.1.6 - v0.1.8)
- **修复**：解决了后端随机遍历导致的题目顺序漂移。
- **增强**：实现级联 `QuickPick` 过滤系统与侧边栏全文搜索。
- **高级特性**：基于 `vscode.diff` 实现了历史代码比对功能（Diff View）。

## 3. 技术难点与解决方案

| 遇到的问题 | 技术原因分析 | 解决方案 |
| :--- | :--- | :--- |
| **切换题目时分栏失效** | `reveal` 方法默认继承当前活动编辑器分栏。 | 强制显式指定 `ViewColumn.Two` 锁定右侧布局。 |
| **提交按钮 ID 绑定错误** | 消息监听器在初始化时形成闭包，未随页面更新同步 ID。 | 将 `problemId` 提升为面板类成员变量，并在 `update` 时强制同步。 |
| **评测进度感知差** | SSE 流推送与 UI 渲染存在延迟感。 | 引入平滑动画进度条，并在完成时通过色彩变化（AC/WA）提供即时反馈。 |

## 4. 结论与后续
Phase 7 标志着系统在交互细节上已达到专业 OJ 插件标准。代码库已建立完善的 Webview 通讯协议，为后续引入协同编码与社交功能打下了坚实基础。
