# PtLPOJ 系统健壮性与安全硬化报告 (Phase 6)

## 1. 对抗性安全测试结果 (Adversarial Testing)

我们针对沙盒环境提交了一系列恶意代码，测试结果如下：

| 测试维度 | 攻击代码描述 | 判定结果 | 判定说明 |
| :--- | :--- | :--- | :--- |
| **文件系统隔离** | 尝试读取 `/etc/passwd` | **WA** | 沙盒环境中缺少该文件或权限受限，输出不匹配，被判定为错误。 |
| **网络隔离** | 尝试通过 `urllib` 访问外部网络 | **TLE** | 网络被彻底禁用，请求阻塞直至超时熔断。 |
| **内存限制** | 大量申请 Python 列表 (>300MB) | **OLE** | 触发 Docker cgroups 限制，进程被内核 SIGKILL 强杀。 |
| **CPU/时间限制** | `while True` 死循环 | **TLE** | 触发 5s 的 Wall-Time 熔断机制，容器被及时销利。 |

## 2. 并发压力测试结果 (Concurrency Test)

使用 `system_test.py` 模拟高压环境：

- **并发量**: 15 个并发 `POST` 请求。
- **请求成功率**: 100% (15/15)。
- **核心稳定性**:
    - **Go Scheduler**: 任务队列平滑进入 `PENDING`，Worker 轮询正常。
    - **SQLite WAL**: 并发落库未出现 `database is locked` 错误。
    - **Docker API**: 容器并发拉起与销毁未见资源竞争冲突。

## 3. 结果总结

系统已通过预设的全部健壮性测试指标。沙盒在资源限制、网络隔离、文件安全方面表现优异；后端处理并发提交时具备良好的背压处理能力。

**结论**：PtLPOJ 系统已达到生产就绪（Production-Ready）的稳定性要求。
